# Release v1.8.1

**Release Date:** 2026-01-24

---

## Improvements

### URL Template Variable Substitution (PR #29, Issue #28)
Enhanced the test executor to support full URL construction from variables in endpoint templates.

- **Full URL Templates**: Endpoints can now contain the complete URL including base URL as variables
  - Example: `{{keyloopUrl}}/{{enterpriseId}}/{{storeId}}/v2/vehicles`
- **Backwards Compatible**: Relative endpoints still work with `apiBaseUrl` variable
- **Missing Variable Detection**: Tests now fail immediately with clear error message listing missing variables
- **Improved Error Messages**: Better guidance when variables are not configured

---

## Technical Changes

### Backend
- Added `substituteVariablesWithTracking()` function to detect missing variables
- Updated URL construction logic to detect full URLs (starting with `http://` or `https://`)
- Fixed warmup request to extract origin from full URL instead of relying on `baseUrl` variable

### URL Resolution Logic
1. Substitute all `{{variable}}` placeholders in endpoint
2. If any variables are missing, fail with descriptive error
3. If result starts with `http://` or `https://`, use as full URL
4. Otherwise, prepend `apiBaseUrl` variable (backwards compatibility)

---

## Example Usage

**Variables:**
```
keyloopUrl = https://api.keyloop.com
enterpriseId = 12345
storeId = 67890
```

**Endpoint Template:**
```
{{keyloopUrl}}/{{enterpriseId}}/{{storeId}}/v2/vehicles
```

**Resulting URL:**
```
https://api.keyloop.com/12345/67890/v2/vehicles
```
