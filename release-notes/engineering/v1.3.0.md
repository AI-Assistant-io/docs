# Release v1.3.0

**Release Date:** 2025-01-24

---

## New Features

### Azure Region Assignment (PR #14, Issue #7)
Clients can now be assigned to Azure regions for future resource provisioning.

- **Azure Regions Table**: New database table storing available Azure regions with abbreviations
- **Region Selection**: Clients can be assigned to a specific Azure region
- **Abbreviated Name**: Clients can have an abbreviated name for Azure resource naming
- **Client Info Display**: Client landing page shows region and Azure resource details
- **Default Region**: UK South is configured as the default region

### Client Landing Page Enhancements
- Displays Azure region assignment
- Shows provisioned resource group and storage account names (when available)
- Improved client info card layout

---

## Technical Changes

- Added `012_add_azure_regions.sql` migration for AzureRegions table with default regions
- Added `013_add_client_region_abbrev.sql` migration adding regionId, abbreviatedName, resourceGroupName, storageAccountName to Clients
- Added `azureRegionController.ts` with CRUD endpoints for Azure regions
- Added `/api/azure-regions` endpoints (GET all, GET default, GET by ID, POST, PUT)
- Updated `clientController.ts` to include region info via LEFT JOIN
- Updated client create/update to support regionId and abbreviatedName
- Updated `ClientLanding.tsx` to display region and Azure resource information
- Updated `api.ts` frontend service with Azure regions methods
