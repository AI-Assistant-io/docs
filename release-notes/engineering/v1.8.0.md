# Release v1.8.0

**Release Date:** 2026-01-24

---

## New Features

### Azure Resource Group Existence Check (PR #26, Issue #24)
Added ability to check and create Azure resource groups for clients with result tracking.

- **Resource Group Naming**: Automatic naming pattern `aia-[region]-rg-[abbreviatedName]`
- **Existence Check**: Verify if a client's resource group exists in Azure
- **Auto-Create**: Optionally create the resource group if it doesn't exist
- **Result Tracking**: All check results stored in database with timestamps
- **History View**: View past resource check results for each client
- **Batch Check**: Check multiple clients' resource groups at once

## Technical Changes

### Backend
- Added `@azure/arm-resources` dependency for Azure Resource Management
- New `azureResourceService.ts` with functions for resource group operations
- New `azureResourceController.ts` with API endpoints
- New migration `017_add_azure_resource_checks.sql` for tracking table
- Added Azure environment variables to `docker-compose.yml`

### Frontend
- Added Azure Resource Group card to `ClientLanding.tsx`
- New API methods in `api.ts` for Azure resource operations

### Database
- New `AzureResourceChecks` table tracking:
  - `clientId` - Link to client
  - `testRunId` - Optional link to test run
  - `resourceType` - Type of resource checked
  - `resourceName` - Full resource name
  - `exists` - Whether resource exists
  - `created` - Whether resource was created
  - `checkedAt` - Timestamp of check
  - `errorMessage` - Any error that occurred

### New API Endpoints
- `GET /api/azure/status` - Check if Azure is configured
- `GET /api/clients/:clientId/azure/resource-info` - Get client's resource group info
- `POST /api/clients/:clientId/azure/check` - Check/create resource group
- `GET /api/clients/:clientId/azure/history` - Get check history
- `GET /api/clients/:clientId/azure/latest` - Get latest check result
- `POST /api/azure/batch-check` - Batch check multiple clients

## Configuration

### Required Environment Variables
```env
AZURE_SUBSCRIPTION_ID=<your-subscription-id>
AZURE_TENANT_ID=<your-azure-ad-tenant-id>
AZURE_CLIENT_ID=<app-registration-client-id>
AZURE_CLIENT_SECRET=<app-registration-secret>
```

### Azure App Registration Requirements
- Create a Service Principal (App Registration) in Azure AD
- Assign **Contributor** role at subscription level
- No redirect URIs needed (daemon/service application)
