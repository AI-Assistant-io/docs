# Release v1.4.0

**Release Date:** 2026-01-23

---

## New Features

### Booking Submission Endpoint (PR #10, Issue #10)
Added a POST endpoint to submit service bookings, simulating integration with an external booking system.

- **Endpoint:** `POST /api/booking`
- **Accepts:** `dueInDateTime`, `dueOutDateTime`, `serviceType`, `notes`
- **Returns:** Booking reference and full payload for debug visibility
- **Logs:** Full booking payload to console (simulates API call)

### Customer Data in Demo Context
Added customer profile data to DEMO_DATA for booking submissions.

- Customer name, email, and ID
- Store and enterprise identifiers
- Advisor and vehicle references

### Vehicle Offers for Finance Customers
Added sample vehicle offers to Finance demo data - similar vehicles at comparable monthly payments to the customer's current car.

- 3 vehicle offers matching the customer's Vortex G1
- Monthly payments ranging from £299-£349 (vs current £326.78)

---

## Improvements

### VHC (Vehicle Health Check) Support in Agent
Updated main agent instructions to understand and handle VHC-related queries.

- Agent now recognises phrases like "when can I have the work done?" and "can I book the repairs?"
- Routes VHC booking requests to the Booking Agent
- Understands context when customer refers to "the work" or "the repairs"
- Added VHC query triggers to the routing guide

---

## Technical Changes

- Added `Customer` section to `DEMO_DATA` with booking-related fields
- Added `vehicleOffers` array to `Finance` demo data
- Added `POST /api/booking` endpoint with simulated API call logging
- Added `datetime` import for booking timestamps
- Added VHC section to main agent instructions
- Added 4 new VHC-related triggers to Query Routing Guide
- Synced updated agent instructions to Azure Foundry
